# Agent `mcpServers` Scoping Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Apply Claude Code v2.1.117's `mcpServers` agent frontmatter to 10 ArcKit agents, scoping each to only the MCP servers it needs; fix the pre-existing govreposcrape allow-list gap in `allow-mcp-tools.mjs` surfaced by this work.

**Architecture:** Linear rollout gated by a verification spike. Spike validates whether the feature applies to Task-tool-spawned agents (ArcKit's pattern) before any rollout. Ten agent files get `mcpServers:` frontmatter; `scripts/converter.py` strips the field for non-Claude targets; `allow-mcp-tools.mjs` gets `mcp__govreposcrape__` added to its allow-list; five canary smoke tests cover all bundled MCP servers before merge.

**Tech Stack:** Claude Code plugin frontmatter (YAML), Python (`scripts/converter.py`), Node.js (hook handler), Markdown-driven agent configuration.

**Spec:** `docs/superpowers/specs/2026-04-22-agent-mcpservers-scoping-design.md`

**Branch:** `feat/agent-mcpservers-scoping` (already created with spec commit)

## Context for Engineers Unfamiliar with ArcKit

Read this once before starting — these facts are load-bearing for the plan.

- **ArcKit** is a plugin for Claude Code that provides 68 architecture-governance slash commands. Source of truth for agents is `arckit-claude/agents/` (ten `.md` files with YAML frontmatter + Markdown body).
- **Plugin agents** run as subagents via Claude Code's **Task tool** when their slash command is invoked (e.g., `/arckit:aws-research` spawns the `arckit-aws-research` agent as a subprocess). This is the "thin-wrapper" pattern.
- **MCP servers** (Model Context Protocol) are declared in `arckit-claude/.mcp.json`. Five are bundled: `aws-knowledge`, `microsoft-learn`, `google-developer-knowledge`, `datacommons-mcp`, `govreposcrape`. Tools from these servers appear to agents as `mcp__<server>__<tool>` (e.g., `mcp__aws-knowledge__search_documentation`).
- **`mcpServers` agent frontmatter** (Claude Code v2.1.117+) lets an agent declare an allow-list of MCP servers it can access. Pre-v2.1.117, all agents saw all MCP servers. The v2.1.117 changelog wording ("loaded for main-thread agent sessions via `--agent`") is ambiguous about Task-tool invocations — Task 1 spike resolves this before rollout.
- **`scripts/converter.py`** transforms the Claude plugin format into Codex/Gemini/OpenCode/Copilot extension formats. Claude-only agent frontmatter fields are listed in `CLAUDE_ONLY_AGENT_FIELDS` at lines 15–20 and stripped by `copy_agent_stripped` (~line 83).
- **Hooks** in `arckit-claude/hooks/hooks.json` fire on Claude Code events. `allow-mcp-tools.mjs` is a `PermissionRequest` handler that auto-approves tool calls from bundled MCPs (users would otherwise see a permission dialog on every MCP tool call).
- **There are no automated tests for agents.** Smoke testing is manual: run the agent in a real test repo and verify it completes and produces its expected artifact.
